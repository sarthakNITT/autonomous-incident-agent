# Debugging Guide

For deep introspection into the agent's behavior.

## Enable Verbose Logging

Set the log level to `debug` in your environment or config.

<Tabs defaultValue="bun">
  <TabsList>
    <div className="flex items-center rounded-[4px] bg-[#18181b] p-0.5 border border-zinc-800/50">
      <TabsTrigger value="bun">bun</TabsTrigger>
    </div>
    <TabsCopyButton />
  </TabsList>
  <TabsContent value="bun">
    ```bash
    LOG_LEVEL=debug bun run start
    ```
  </TabsContent>
</Tabs>

This will output detailed info about:
-   OTel span attributes.
-   LLM prompts and responses.
-   Git command outputs.

## Inspecting Artifacts (R2)

AIA stores its "thought process" in R2. Inspecting these files is the best way to understand *why* the agent made a decision.

1.  **Locate the Run ID**: Check the logs for `Run ID: <uuid>`.
2.  **Download the Context**: Get `runs/<uuid>/context.json` from your bucket.
3.  **Check the Autopsy**: Download `autopsy/<incident_id>/report.md`.

## Manual Reproduction

If the agent fails to reproduce an bug, you can try running the generated script manually.

1.  Download `repro.ts` from R2 (`patches/<incident_id>/repro.ts`).
2.  Place it in your local repository.
3.  Run it with `bun test repro.ts`.

If it fails or passes unexpectedly, you can debug the test case directly.
