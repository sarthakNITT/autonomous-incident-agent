# Running the Agent

This guide details how to run the AIA Agent service in production and development.

## Production (Bun)

The most performant way to run the agent is using Bun.

<Tabs defaultValue="bun">
  <TabsList>
    <div className="flex items-center rounded-[4px] bg-[#18181b] p-0.5 border border-zinc-800/50">
      <TabsTrigger value="bun">bun</TabsTrigger>
    </div>
    <TabsCopyButton />
  </TabsList>
  <TabsContent value="bun">
    ```bash
    bun run start
    ```
  </TabsContent>
</Tabs>

This command starts the `Agent`, `Router`, and `Autopsy` services according to your `aia.config.ts`.

### Environment Variables

Ensure the following are set in your production environment:

*   `GITHUB_TOKEN`
*   `R2_*` credentials
*   `AIA_API_KEY` (for the LLM)

## Development

For contributing to AIA or debugging local issues:

<Tabs defaultValue="bun">
  <TabsList>
    <div className="flex items-center rounded-[4px] bg-[#18181b] p-0.5 border border-zinc-800/50">
      <TabsTrigger value="bun">bun</TabsTrigger>
    </div>
    <TabsCopyButton />
  </TabsList>
  <TabsContent value="bun">
    ```bash
    bun run dev
    ```
  </TabsContent>
</Tabs>

This starts the services with hot-reloading enabled. Logs will be streamed to stdout in a human-readable format.

## Docker

You can also run the agent as a Docker container.

<Tabs defaultValue="docker">
  <TabsList>
    <div className="flex items-center rounded-[4px] bg-[#18181b] p-0.5 border border-zinc-800/50">
      <TabsTrigger value="docker">docker</TabsTrigger>
    </div>
    <TabsCopyButton />
  </TabsList>
  <TabsContent value="docker">
    ```bash
    docker run -d \
      --env-file .env \
      -p 3001:3001 \
      ghcr.io/check19-studios/aia-agent:latest
    ```
  </TabsContent>
</Tabs>
