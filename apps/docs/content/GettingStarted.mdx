## Quick Start (As a Service User)

If you just want to monitor your Vercel or GitHub Pages deployments:

<Steps>
  <Step>
    ### Sign Up

    Visit the AIA dashboard and create an account (powered by Clerk).
  </Step>

  <Step>
    ### Create a Project

    Click **New Project** and fill in:

    | Field | Description |
    | :--- | :--- |
    | **Project Name** | A friendly name for your app |
    | **Repository URL** | `https://github.com/org/repo` |
    | **GitHub Token** | Personal Access Token (for auto-PR) |
    | **Base Branch** | Usually `main` or `master` |
    | **Resolution Mode** | **Auto PR** (create PRs) or **Manual Fix** (show steps) |

    AIA auto-detects your available branches from GitHub.
  </Step>

  <Step>
    ### Add the Webhook

    Copy the webhook URL from your project card and paste it into:

    - **Vercel**: Settings â†’ Git â†’ Deploy Hooks

## Core Concepts

### Projects

A **Project** links your GitHub repository to AIA. Each project has:
- Its own GitHub token (per-project credential isolation)
- A base branch for auto-PRs
- A resolution mode (Auto PR or Manual Fix)
- A unique webhook URL

### Incidents

An **Incident** is created whenever a deployment fails. It tracks:
- The error details
- AI-generated root cause analysis
- Fix suggestions (prompt, patch diff, manual steps)
- Current lifecycle status

### Resolution Modes

| Mode | What happens |
| :--- | :--- |
| **Auto PR** | AIA creates a GitHub PR with the AI-generated fix |
| **Manual Fix** | Dashboard shows root cause, AI prompt, and manual steps |
