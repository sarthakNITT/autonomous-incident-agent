# Incident Timeline

## Overview

The timeline answers the question: *"Where exactly is this incident in the resolution process?"*

Each incident has a **5-stage progress bar** that tracks its journey from detection to resolution, making it easy to see at a glance what's happening across all your deployments.

## Accessing the Timeline

Click **Timeline** in the navigation, or go directly to `/timeline`. Sign-in is required.

## Features

### Real-Time Auto-Refresh

The timeline polls `/api/incidents` every **10 seconds** automatically. A **Live** indicator (pulsing bell) confirms the feed is active.

### Status Filters

Filter incidents by lifecycle stage using the pill buttons:

```
[All (12)] [Detected] [Analyzing] [Patching] [Resolved] [Failed]
```

## API

The timeline reads from:

```
GET /api/incidents
```

This returns all incidents for the authenticated user, including status, timestamps, repo info, and PR links.
