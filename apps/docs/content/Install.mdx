# Installation

This guide covers the installation and setup of the Autonomous Incident Agent (AIA) using Bun and TypeScript.

## Prerequisites

Before proceeding, ensure your environment meets the following requirements:

*   **Runtime**: [Bun](https://bun.sh) v1.0 or later.
*   **Node.js**: v20 or later (if using Node-based tools).
*   **Storage**: A Cloudflare R2 bucket for storing artifacts (autopsy reports, patches).
*   **Version Control**: A GitHub repository where the Agent can open Pull Requests.

## System Requirements

### Hardware

*   **CPU**: 2+ cores recommended for concurrent agent operations.
*   **Memory**: 4GB+ RAM (Agent analysis can be memory intensive).
*   **Disk**: Fast I/O is recommended for log processing and temporary workspace creation.

### Network

The Agent requires outbound access to:

*   **GitHub API**: `api.github.com`
*   **Cloudflare R2**: `S3 Compatible Endpoint`
*   **LLM Provider**: Your configured AI provider API (e.g., OpenAI, Anthropic).

## Configuration

After installation, you must configure the agent using `aia.config.ts`. See the [Configuration Reference](./ConfigReference.mdx) for details.

## Troubleshooting

If you encounter issues during installation:

1.  Ensure `bun` is in your system PATH.
2.  Check for write permissions in the directory (the agent creates a `.agent` folder).
3.  Verify your network allows connections to npm/GitHub packages.

See [Troubleshooting](./Troubleshooting.mdx) for more solutions.
